# Comment Toxicity Detection 💬

A deep learning project that classifies text comments as **toxic** or **non-toxic** using an LSTM neural network, with a Streamlit web app for interactive predictions.

---

## Overview

This project trains a binary text classifier on labeled comment data to detect online toxicity. It includes:

- Text preprocessing and tokenization
- An LSTM-based deep learning model built with TensorFlow/Keras
- Model evaluation using classification metrics
- A Streamlit web app for single comment and bulk CSV predictions
- Model and tokenizer export for reuse

---

## Project Structure

```
├── Toxicity.ipynb        # Main notebook (training + app)
├── app.py                # Streamlit web application (generated by notebook)
├── toxicity_model.h5     # Saved Keras model (generated after training)
├── tokenizer.pkl         # Saved tokenizer (generated after training)
└── train.csv             # Training dataset (user-provided)
```

## Dataset

The notebook expects a CSV file at `/content/train.csv` with at least the following columns:

| Column         | Description                        |
|----------------|------------------------------------|
| `comment_text` | Raw text of the comment            |
| `toxic`        | Binary label: `1` = toxic, `0` = non-toxic |

> The dataset is read with `latin1` encoding and bad lines are skipped automatically.

---

## How to Run

### 1. Train the Model (in Google Colab or Jupyter)

Open `Toxicity.ipynb` and run all cells in order:

1. Install dependencies
2. Load and preprocess the dataset
3. Tokenize and pad sequences
4. Train the LSTM model (10 epochs, batch size 32)
5. Evaluate with classification report
6. Save `toxicity_model.h5` and `tokenizer.pkl`

### 2. Launch the Web App

The notebook writes `app.py` and starts a Streamlit server:

```bash
streamlit run app.py
```

In Google Colab, the notebook uses `localtunnel` + `pyngrok` to expose the app publicly.

## Model Architecture

| Layer       | Details                              |
|-------------|--------------------------------------|
| Embedding   | `input_dim=20000`, `output_dim=128`  |
| LSTM        | 64 units                             |
| Dropout     | 50%                                  |
| Dense       | 32 units, ReLU activation            |
| Output      | 1 unit, Sigmoid activation           |

- **Loss**: Binary Cross-Entropy  
- **Optimizer**: Adam  
- **Metric**: Accuracy  

---

## 🖥️ Web App Features

### Single Comment Prediction
- Enter any comment in the text area
- Click **Predict** to get a toxicity score (0–1)
- Score > 0.5 → 🚨 **Toxic** | Score ≤ 0.5 → ✅ **Non-Toxic**

### Bulk CSV Prediction
- Upload a CSV file with a `comment_text` column
- The app scores all comments and labels each one
- Download results as a CSV with `toxicity_score` and `label` columns

---

## Evaluation

After training, the model is evaluated on a 20% test split using `classification_report` from scikit-learn, which includes precision, recall, and F1-score for both classes.

---

## Notes

- Text is cleaned by lowercasing, removing non-alphabetic characters, and stripping extra whitespace.
- Sequences are padded/truncated to a max length of **100 tokens**.
- The tokenizer vocabulary is capped at **20,000 words**.
- The notebook is designed to run on **Google Colab** but can be adapted for local use by updating file paths.
